- Clinically annotated multi-pathology gait dataset with **multi-IMU** recordings.
- Typical characteristics: multiple conditions, repeated trials, standardized walking protocols.

------

- How robust are gait ML models across **pathologies**, **conditions**, and **sensor availability**?

------

- **Required:** pathology / impairment classification (multi-class).
- **Required:** **robustness benchmarking** (leave-one-group-out: subject / condition / pathology).

- Reproducible pipeline: preprocessing, splits, baselines, metrics, ablations.

## 课程的 4 个核心目标（这是“评分逻辑”）

### 🎯 目标 1：你是否理解 *Biomedical ML 问题本身*

这门课首先关心的不是模型，而是：

- 你是否理解 **数据来自哪里**
- 数据有没有 **临床/生理意义**
- 标签是不是“干净的”
- 任务设定是否合理

👉 在 gait 项目里具体体现为：

- 使用 **多病种 / 临床采集的 IMU 步态数据**
- 理解：
  - gait ≠ 普通 time series
  - 病理 gait 的 variability 非常大
- 明确问题：
  - classification / phenotyping / pathology recognition
  - 而不是“随便跑个 CNN”

✅ **这是“问题建模能力”的考察**

------

### 🎯 目标 2：你是否会做 *Robustness / Generalization* 分析（重点）

这是**这门课最重要的目标**。

老师要你回答的是：

> 一个看起来 performance 很高的模型，
> 在现实条件变化下，还靠谱吗？

具体体现在你必须分析：

- 跨病种（cross-pathology）
- 跨实验条件（trial / speed / protocol）
- 跨传感器配置（sensor dropout / reduced IMUs）

👉 所以 slide 里反复强调：

- robustness
- sensor availability
- pathology variability

⚠️ **不是为了折磨你，是为了逼你做“真实世界 ML”**

------

### 🎯 目标 3：你是否具备 *严谨的 ML 实验设计能力*

这门课非常在意你有没有做到：

- 合理的 train / validation / test split
- 避免 data leakage
- 有对照实验（baseline vs ablation）
- 有统计分析，而不是只报 accuracy

比如在 gait 项目中：

- 全 IMU vs 少 IMU
- 单病种训练 vs 多病种训练
- Subject-wise split（而不是 window-wise）

👉 **不在乎你模型多 fancy，在乎你实验干不干净**

------

### 🎯 目标 4：你是否能做 *科学叙事（Scientific Storytelling）*

这是研究生课程的隐藏目标。

老师希望你能：

- 把一个工程实验

包括：

1. 清楚的问题动机
2. 明确的研究问题（Research Question）
3. 有逻辑的实验设计

你现在被要求做的，其实是：

> **一个 mini benchmark / evaluation study**

而不是 algorithm paper。

## 这门课对 Project 的**明确要求**

### 1️⃣ 必须使用指定的 **真实生物医学数据**

- 使用 **真实采集的 biomedical 数据**（不是 toy / synthetic）
- 数据应具有：
  - 多受试者（multi-subject）
  - 生理 / 临床意义（如 gait、IMU、pathology）
- 数据来源、标签含义、采集条件必须说明清楚

------

### 2️⃣ 必须提出一个**明确、可检验的研究问题**

- 不是“跑模型看效果”
- 而是一个**关于模型行为的问题**，例如：
  - 模型在条件变化下是否稳定？
  - 模型是否能泛化到未见病种 / 条件？
- 问题必须：
  - 清晰
  - 可通过实验回答

------

### 3️⃣ 必须评估 **模型的鲁棒性 / 泛化能力（核心）**

课程**强制要求你分析 robustness**，至少包含以下之一（通常不止一个）：

- 跨病种（cross-pathology）
- 跨实验 / 行为条件
- 传感器变化（如 sensor dropout / reduced IMUs）

⚠️ **只报告一个“最优条件下的 accuracy”是不合格的**

------

### 4️⃣ 必须有 **系统性的实验设计**

包括但不限于：

- 合理的数据划分（避免 data leakage）
- 至少一个 baseline
- 至少一个 ablation / 对照实验
- 清楚说明：
  - train / test 如何分
  - 每个实验在比较什么

------

### 5️⃣ 必须使用**合适的评价指标和分析**

- 不只一个指标（不只 accuracy）
- 结果要：
  - 可比较
  - 可解释
- 鼓励（但不强制）：
  - 统计分析
  - 结果趋势解释

------

### 6️⃣ 不要求你提出新算法（但允许）

- ❌ 不要求 method innovation
- ❌ 不以模型复杂度评分
- ✅ 更看重：
  - 问题定义
  - 实验严谨性
  - 结论是否可信

------

### 7️⃣ 最终成果必须是**科研风格的报告**

- 清楚的结构：
  - Introduction / Motivation
  - Methods
  - Experiments
  - Results
  - Discussion
- 能回答你一开始提出的研究问题
- 逻辑自洽、可复现
